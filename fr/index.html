Comment construire et publier une documentation Sphinx sur GitHub Pages en utilisant Drone CI#

Pour suivre ce tutoriel vous devez avoir:

  • un dépôt GitHub contenant une documentation Sphinx sitée dans le répertoire documentation à la racine du dépôt,

  • un fichier documentation/requirements.txt contenant les dépendances Python,

  • un fichier documentation/conf.py contenant la configuration Sphinx.

Initialiser la branche gh-pages#

La branche gh-pages va contenir la documentation générée. On doit la créer et y ajouter quelques fichiers:

cd /path/to/repo

# Create an orphan branch (no parent commit) and remove all files
git checkout --orphan gh-pages
git rm -rf .

# Tell Github to not try to build our documentation
touch .nojekyll

# Optional, but can be useful to have links to different versions of the
# documentation (different languages for instances)
touch index.html

git add .nojekyll index.html
git commit -m "Initial commit"
git push origin gh-pages

À titre d’exemple, vous pouvez mettre le contenu suivant dans index.html:

<!DOCTYPE html>
<html>
    <head>
        <title>MyRepo user manual</title>
    </head>
    <body>
        <h1>MyRepo user manual</h1>

        <ul>
            <li><a href="en/">English</a></li>
            <li><a href="fr/">French</a></li>
        </ul>
    </body>
</html>

Créer un miroir sur git.biblibre.com#

Puisque notre instance Drone fonctionne seulement sur git.biblibre.com, on doit créer un dépôt miroir ici.

  1. Allez sur https://git.biblibre.com

  2. Cliquez sur le signe + dans le coin supérieur droit puis Nouvelle migration

  3. Choisir GitHub

  4. Saisir l’URL du dépôt et cocher Ce dépôt sera un miroir

  5. Choisir les bons propriétaire et nom de dépôt

  6. Cliquer sur le bouton Migrer le dépôt

Créer une clé de déploiement GitHub#

Drone va avoir besoin de pousser dans le dépôt GitHub. On a besoin d’une clé de déploiement pour ça. C’est une clé SSH qui aura un accès en écriture seulement pour ce dépôt.

  1. Generate an SSH keys pair with no passphrase:

    ssh-keygen -f /tmp/github_deploy_key -t ed25519 -C drone+myrepo@biblibre.com

  2. Aller sur la page des paramètres du dépôt GitHub et cliquer sur Deploy keys (ou aller directement sur https://github.com/<USER>/<REPO>/settings/keys)

  3. Cliquer sur Add a deploy key

  4. Saisir un titre

  5. Copier la clé publique dans la zone de texte

  6. Cocher Allow write access

Configurer Drone#

  1. Aller sur https://drone.biblibre.com

  2. Cliquer sur SYNC

  3. Une fois la synchronisation terminée, chercher le module et cliquer sur Activate

  4. Dans les paramètres, cocher Disable pull requests et Disable forks

  5. Cliquer sur Save

  6. Dans la section Secrets, saisir GH_DEPLOY_KEY dans le champ Secret Name, et copier la clé privée dans le champ Secret Value.

  7. Laisser la case à cocher Allow pull requests décochée.

  8. Cliquer sur Add a secret.

Ajouter la configuration Drone au dépôt#

Dans la branche principale du dépôt, ajouter les fichiers suivants:

.drone.yml#
---
kind: 'pipeline'
type: 'docker'
name: 'documentation'
steps:
  - name: 'build'
    image: 'python:3'
    commands:
      - 'sh .drone/documentation-build.sh'
  - name: 'push'
    image: 'alpine'
    commands:
      - 'apk add git openssh'
      - 'sh .drone/documentation-push.sh'
    environment:
      GH_DEPLOY_KEY:
        from_secret: 'GH_DEPLOY_KEY'
trigger:
  branch:
    - 'master'
  event:
    - 'push'
.drone/documentation-build.sh#
#!/bin/sh

cd "$(dirname -- "$(dirname -- "$(readlink -f -- "$0")")")/documentation"
pip install -r requirements.txt
languages="en fr"
for language in $languages; do
    make -e BUILDDIR=_build/$language SPHINXOPTS="-D language=$language" clean html
done
.drone/documentation-push.sh#
#!/bin/sh

unset GIT_AUTHOR_NAME
unset GIT_AUTHOR_EMAIL
unset GIT_AUTHOR_DATE
unset GIT_COMMITTER_NAME
unset GIT_COMMITTER_EMAIL
unset GIT_COMMITTER_DATE

git config --global user.email "drone@biblibre.com"
git config --global user.name "Drone CI"

mkdir -p ~/.ssh
printenv GH_DEPLOY_KEY > ~/.ssh/deploy_key
chmod 600 ~/.ssh/deploy_key
cat > ~/.ssh/config << 'CONFIG'
Host github.com
User git
IdentityFile ~/.ssh/deploy_key
StrictHostKeyChecking accept-new
CONFIG

cd "$(mktemp -d)"
git clone --branch gh-pages git@github.com:<USER>/<REPO>.git .

languages="en fr"
for language in $languages; do
    rm -rf $language
    cp -r "$DRONE_WORKSPACE/documentation/_build/$language/html" $language
    git add $language
done
git commit -m "Drone build: $DRONE_BUILD_NUMBER" -m "Triggered-by: $DRONE_COMMIT_SHA"

git push origin gh-pages

Avertissement

Ne pas oublier de remplacer <USER> et <REPO> par les bonnes valeurs

Rendre les scripts exécutables:

chmod +x .drone/*.sh

Puis valider et pousser:

git add .drone.yml .drone/*.sh
git commit -m 'Add Drone files to automatically build documentation'
git push origin master

Tester!#

Tout est prêt et la documentation devrait être automatiquement générée quand le miroir sur git.biblibre.com sera synchronisé (une fois toutes les 8 heures par défaut).

Mais il est possible de tester tout de suite en déclenchant manuellement la synchronisation. L’option se trouve dans les paramètres du dépôt miroir.
